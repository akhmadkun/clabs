# Layer 3 + Inter LAN Connections
   + Make sure clients can ping their gateway
   + Make sure client1 can ping client2
   + Make sure client2 can ping client1
   + You are only allowed to create 1 security policy



# Layer 2 + SVI


   + Make sure client3 can ping client4
   + Create svi gateway for this vlan (192.168.34.1)
   + Make sure this clients can reach their gateway
   + Make sure this clients can reach Client1 and Client2



# Source NAT


   + Configure ethernet1/5 with IP 203.0.113.1/30
    - Assign it to a new zone named Untrust

   + Configure a Default Route (0.0.0.0/0) in the Virtual Router
    - Set next-hop to 203.0.113.2

   + Create a Source NAT Policy named Internet-Access
    - Translate traffic from LAN1, LAN2, and LAN34 to Untrust using the IP of eth1/5

   + Create a Security Policy to Allow Traffic from Internal Zones to Untrust
   + Verify that Client1 can ping 8.8.8.8, 8.8.4.4



# Destination NAT (DMZ & Port Forwarding)


    1. Configure ethernet1/6 with IP 10.0.0.1/24 and assign it to a new zone named DMZ


    2. Create a Destination NAT Policy named Inbound-Web
       If traffic from Untrust is destined to 203.0.113.1 (public IP on ethernet1/5),
       translate the destination to the internal Web Server IP 10.0.0.100


    3. Create a Security Policy to Allow Inbound Traffic
       IMPORTANT:
       - name : Allow-Inbound-Web
       - Source Zone  = Pre-NAT Zone (Untrust)
       - Destination Zone = DMZ (zone after routing decision)
       - Destination IP   = Post-NAT IP (10.0.0.100)

    4. (Advanced Simulation)
       Run a mini Python web server on the cEOS Web_Server node,
       then test access from ISP_Router using:
       curl 203.0.113.1
